💻 Master 1 | First Semester Project

This web application is Integrated Patient Health System (IPHS) designed to support healthcare professionals, researchers, and students in analyzing patient data, biomedical signals, and medical images.

The application combines data management, statistical analysis, signal processing, image processing, and visual analytics within a single, intuitive graphical user interface (GUI).

Login Page

Users are greeted with a secure login page requiring valid credentials to access the system. Upon successful authentication, users are directed to the main dashboard.

Main Window Structure

The application window is divided into three primary areas:

Sidebar Navigation (Left Panel)

The sidebar provides quick access to all major functionalities:

  • Patient Data Management
  • Health Data Analysis
  • Spectrum Analysis
  • Medical Image Processing

Functionalities Content Area

This area dynamically displays:

  • Data tables
  • Charts and plots
  • Signal waveforms
  • Medical images
  • Analysis results

Each feature opens within its own dedicated tab for clarity and workflow separation.

🛠️ Functionalities Tabs and How to Use (Navigate)

📂 Data Loading and Management

This functionality tab handles data ingestion, storage, and database operations.

Loading Data

Users can load data from:

  • Local files (CSV, Excel)
  • Databases (SQL)

Once loaded, data is displayed in interactive tables with options to sort, filter, and search records.

Managing Data

Users can add, edit, or delete patient records directly within the application. Changes can be saved back to the database to ensure data integrity.

Steps:

  1. Navigate to the "Patient Data Management" tab via the sidebar.
  2. Select "Load Data" to import datasets.
  3. Use the table interface to manage records as needed.

Available actions:

  • Insert New DataAdd new patient records or image metadata
  • Retrieve DataFetch existing records from the database
  • Update Data Modify selected patient or image records
  • Delete DataRemove selected records

Each action provides confirmation messages or error prompts where applicable.

Table View

The system provides a robust data display with the following features:

  • Displays loaded or retrieved data
  • Supports scrolling for large datasets
  • Enables row selection for update or deletion

This tab focuses on statistical and exploratory analysis of patient health metrics.

Data Filtering

Users can:

  • Select health variables (e.g., heart rate, blood pressure)
  • Apply filters such as: Moving average, Noise reduction, Outlier removal
  • Adjust parameters using sliders

Steps:

  1. Select two health metrics from dropdown menus
  2. Click Compute Correlation
  3. View: Correlation coefficient, Scatter plot, and Correlation heatmap

This helps identify relationships between physiological parameters.

  • Toggle between raw and filtered signals
  • Useful for trend analysis and anomaly detection

This tab enables frequency-domain analysis of biomedical signals.

Signal Loading

Users can:

  • Select ECG, EEG, or other biomedical signals
  • Raw signal waveform is displayed immediately

FFT Spectrum Analysis

Steps:

  • Select a signal
  • Choose FFT parameters (e.g., window size)
  • Add Noise (Gaussian, etc)
  • Click Compute FFT
  • View: Frequency spectrum | Power spectrum

Sliders allow selection of specific time windows or signal segments.

This tab supports image processing such as enhancement, segmentation, grayscale, and many more for medical imaging.

Image Loading

  • Upload X-ray, MRI, or CT images
  • Original image appears on the middle panel
  • Processed image appears on the right panel

The functionality buttons allow users to apply various image processing techniques.

Image Processing Operations

Available operations include:

  • Grayscale Conversion
  • Smoothing / Blurring
  • Edge Detection (Canny)
  • Image Difference
  • Difference
  • Thresholding
  • Histogram Equalization

Processed images appear on the right for side-by-side comparison. There is also a "Save" button to save the processed image.

  • One-click switching between modules
  • Active section is visually highlighted

Dynamic Controls

  • Controls update automatically based on selected data
  • Analysis options are context-aware

Reset and Error Handling

  • Data fails to load
  • Database operations fail
  • Invalid inputs are selected
  • Reset buttons clear selections and plots
  • Error messages guide users to resolve issues

Consistent Theming and Color Scheme

  • Uniform colors and fonts across all tabs
  • Enhances user familiarity and comfort
  • Color cues: Green = successful actions | Red = errors | Yellow = warnings

Accessibility Features

  • Tooltips on hover for buttons and controls
  • Responsive layout for different screen sizes

Intuitive Layouts

  • Logical grouping of related controls
  • Minimizes cognitive load
  • Clear labels and instructions

Performance and Reliability

  • Optimized data handling for large datasets
  • Robust error handling to prevent crashes
  • Non-blocking FFT and image operations
  • Optimized memory usage for large images and tables

These design choices ensure a smooth and efficient user experience.

Remarks

The application has been extensively tested for stability.
